# Contributing to the Architectural Portfolio

Thank you for your interest in contributing to this project! We welcome contributions that improve the user experience, add new features, or fix bugs.

## 🛠️ Development Workflow

### 1. Fork & Clone
- Fork the repository and clone it to your local machine.
- Create a new branch for your feature or bugfix:
  ```bash
  git checkout -b feature/your-feature-name
  ```

### 2. Code Style
- Follow the existing project structure and naming conventions.
- Use **TypeScript** for all new code.
- Use **Tailwind CSS** for styling.
- Ensure all components are functional and use React hooks.
- Use **motion/react** (Framer Motion) for animations.

### 3. Commit Messages
- Use clear and descriptive commit messages.
- Reference issue numbers if applicable.

### 4. Pull Requests
- Provide a clear description of the changes in your pull request.
- Include screenshots or videos for UI changes.
- Ensure all tests and lint checks pass before submitting.

## 🎨 Design Guidelines

- **Minimalism**: Maintain a clean, high-contrast aesthetic.
- **Typography**: Prioritize legibility and hierarchy.
- **Interactions**: Use subtle animations to guide the user's attention.
- **Responsiveness**: Ensure a premium experience on both mobile and desktop.

## 🔒 Security & Privacy

- Never commit API keys or secrets to the repository.
- Use environment variables for sensitive configuration.
- Follow Firebase Security Rules best practices when modifying data access.

## 🧪 Testing

- Test your changes thoroughly on different screen sizes and browsers.
- Verify that project filtering and search work as expected.
- Ensure the admin dashboard correctly saves and updates project data.

Thank you for helping us build a better portfolio!
